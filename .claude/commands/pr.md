Создать PR(s) для текущей работы.

1. Определить затронутые репо — проверить git log и git diff в каждом.

2. Для каждого репо с изменениями:
   - Составить заголовок PR (conventional, до 70 символов)
   - Написать описание: что сделано, почему, как проверить
   - Создать PR через mcp__github__create_pull_request (org: SilentCare-so, base: main)

3. Для cross-repo задач (изменения в нескольких репо):
   - Создать отдельный PR в каждом репо
   - В description каждого PR добавить ссылку на связанный PR в другом репо

4. Вернуть ссылки на все созданные PR.

Никогда не использовать `gh` CLI.
